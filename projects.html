Ochlo

UX Design Security

Approaching the challenge of creating a secure, decentralized communication tool, I designed Ochlo. This project focused on user privacy and ease of use, allowing for the instant creation of encrypted WLAN events without requiring user accounts.

2 weeks
Timeline
Solo
Team Size
Mobile
Platform
View on Behance

Problem Statement

Current communication tools require account creation and centralized servers, creating privacy concerns and barriers to quick, secure communication in sensitive situations.

Key Challenges

  • Balancing security with ease of use
  • Creating intuitive onboarding without accounts
  • Designing for various technical skill levels

Solution Approach

I designed a streamlined interface that generates secure communication channels instantly, using visual metaphors that make complex security concepts accessible to all users.

Key Features

  • One-tap event creation
  • Visual encryption status indicators
  • Progressive disclosure of advanced features

Eugene Access

Public Service Accessibility

As a one-day personal design challenge, I created Eugene Access to solve a key usability problem: quickly finding essential human services. The focus was on a simple, single-screen interface for maximum accessibility and efficiency.

1 day
Timeline
Solo
Team Size
Web
Platform
View on Behance

Problem Statement

People in crisis situations struggle to quickly find essential services due to fragmented information sources and complex navigation systems in existing platforms.

Design Constraints

  • 24-hour design challenge
  • Single-screen interface requirement
  • Accessibility-first approach

Design Philosophy

Prioritized clarity and speed over visual complexity. Every element serves the core purpose: connecting people to help as quickly as possible.

Key Decisions

  • Large, high-contrast buttons
  • Icon + text labeling system
  • Emergency services prominently featured

Cash Cache

FinTech Data Viz

The goal for Cash Cache was to make personal finance more engaging. I designed this income visualizer to provide clear, graphical representations of finances, turning a mundane task into a more interactive and satisfying experience.

3 weeks
Timeline
Solo
Team Size
Mobile
Platform
View on Behance

Problem Statement

Traditional financial apps present data in boring, text-heavy formats that fail to engage users or provide meaningful insights into spending patterns.

Research Insights

  • Users avoid checking financial apps
  • Visual data is processed 60x faster than text
  • Gamification increases engagement

Visual Strategy

Created an engaging visual language using progressive data visualization that reveals insights gradually, making complex financial data approachable and actionable.

Impact Goals

  • Increase user engagement with finances
  • Make budgeting feel rewarding
  • Simplify complex financial concepts
